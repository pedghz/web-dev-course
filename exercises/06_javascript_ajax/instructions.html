2. Javascript PostMessage

This exercises involves using .postMessage() to send messages from an iframe to the parent document. You are to write a simple 'service' in reverser_service.html that takes a string, reverses it and sends it as a message to the document that contains reverser_service.html as an iframe.

user_actions.html is given to you so you can test your solution with that. Note that you should not rely on any modifications on user_actions.html, the grading will be done using the given file. The messages should have a payload attribute that contains:

  1. ... either the string that was typed into #toBeReversed reversed if the #reverser form was submitted.
  2. ... or 'INPUT CLEARED' if the #clearInput was clicked.
	//Example message
	msg = {};
	msg.payload = 'elloH'; // (1) #toBeReversed contained 'Hello' and user submitted #reverser
	msg.payload = 'INPUT CLEARED' // (2) User clicked #clearInput

Note that you can assume that the grader will also have 'lib/jquery-1.11.3.min.js'.

Where to Start

  • Read about postMessage from MDN.
  • Create reverser_service.html and create the necessary elements.
  • Write the JavaScript code that handles the user actions and sends messages to the window.parent. Write the code inside the HTML file.

3. JSONP Currency Exchange Rates

Fixer.io (http://fixer.io/) offers a JSONP API for exchange rates for different currencies. Your task is to write a simple page, which shows exchange rates for a given date in the available currencies.

Static parts

Start by creating new currency.html and currency.js files. Create an empty HTML5 page and include the jQuery library and your own JavaScript file after it. For grading you must include the jQuery like this: 

<script src="lib/jquery-1.11.3.min.js"></script>

Next, add a form element which contains an input type=date element with id=date and input type=submit element with id search. This form is used to query the feed with the entered date. The form can look for example like in the image below. Furthermore, add an empty table element with id currencies for adding the exchange rates later on.

JSONP request

Note that you need to always wait for the HTML DOM to be completely loaded and prepared before looking for any DOM elements. Check jQuery examples for document ready.

Now using jQuery, select the form and attach an event listener to submit event. In the handler function remember to prevent the default action so that the page is not reloaded when the form is submitted. You can use console.log("remarks", somedata); to debug how your code works.

You can query the Fixer.io JSONP API for a specific date using following URL structure:
http://api.fixer.io/2010-10-20 where the last part is YYYY-MM-DD.

To build the request URL get the entered tag value and URL-encode it. See exercise 1 questionnaire for making the JSONP request. Some useful functions: on, event.preventDefault, val, jQuery.getJSON

Dynamic DOM elements

At this point you should be able to receive a data object requested from the Fixer.io feed and for example log the contents. Let the received feed object be in a variable data. The exchange rates are in an object data.rates that you should loop through. In the data.rates object there are value pairs for each available currency code and the exchange rate for that currency. E.g. "AUD": 1.4151.

Next, you will populate the table#currencies so that each row has two cells, where in the first one is the currency code and on the second one is the exchange rate for the queried date.

You do not have to handle errors in the API requests. However, you should empty the old table before querying for new data.
